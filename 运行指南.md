## 常见问题与故障排除 (FAQ)

### 🐍 1. Python 环境与依赖问题

#### Q1: 运行 `pip install -r requirements.txt` 时提示 `Could not find a version that satisfies the requirement torch==...`
- **原因**：PyTorch 版本与你的操作系统、Python 版本或 CUDA 驱动不兼容。
- **解决方案**：
    1. 前往 [PyTorch 官网](https://pytorch.org/get-started/locally/) 根据你的环境生成正确的安装命令。
    2. **CPU 环境（推荐新手）**：`pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu`
    3. **边缘设备（ARM架构）**：需安装 PyTorch ARM 版本：`pip install torch --index-url https://download.pytorch.org/whl/cpu`（需确认 ARMv7 或 aarch64）。

#### Q2: 运行 `python app.py` 报错 `ModuleNotFoundError: No module named 'flask_cors'`
- **原因**：依赖包未完全安装。
- **解决方案**：手动安装缺失的包：`pip install flask-cors`

#### Q3: 虚拟环境激活失败（Windows PowerShell 报错 `无法加载文件 ...\Activate.ps1`）
- **原因**：PowerShell 执行策略限制。
- **解决方案**：
    1. 以管理员身份打开 PowerShell。
    2. 输入命令：`Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`
    3. 输入 `Y` 确认，然后重新激活环境。

### 🚄 2. 模型训练与量化问题

#### Q4: 运行 `train.py` 报错 `FileNotFoundError: [Errno 2] No such file or directory: '.../dataset/...'`
- **原因**：数据集路径配置错误。
- **解决方案**：修改 `backend/model/train.py` 中的 `data_dir` 变量，确保指向你解压后的数据集文件夹根目录（如 `'../data/UCMerced_LandUse/Images'`）。

#### Q5: 训练速度极慢 / 显存不足 (CUDA out of memory)
- **原因**：设备算力不足或 Batch Size 设置过大。
- **解决方案**：
    1. 在 `train.py` 中将 `batch_size` 从默认的 `32` 改为 `16` 或 `8`。
    2. 如果电脑没有 NVIDIA 显卡，训练会自动切换为 CPU 模式，速度会很慢，建议仅做演示测试。

#### Q6: 运行 `quantize.py` 报错 `No module named 'torch.quantization'`
- **原因**：PyTorch 版本过低（需 >= 1.3）。
- **解决方案**：升级 PyTorch：`pip install --upgrade torch torchvision`

### 🌐 3. 后端 API 运行问题

#### Q7: 运行 `app.py` 提示 `Address already in use` 端口被占用
- **原因**：5000 端口已被其他程序（如 AirPlay、迅雷、其他 Flask 进程）占用。
- **解决方案**：
    1. **方法一（推荐）**：修改 `app.py` 最后一行代码：`app.run(host='0.0.0.0', port=5001, debug=True)`，将 `5000` 改为其他端口如 `5001`、`8081`。注意：改了端口后，前端代码也要对应修改。
    2. **方法二**：Windows 下查找占用端口的进程并结束：`netstat -ano | findstr :5000` -> 记住 PID -> `taskkill /PID [PID号] /F`

#### Q8: 前端请求后端报错 `CORS policy` 跨域错误（浏览器控制台红字）
- **原因**：前端页面打开的地址（如 `http://127.0.0.1:8080`）与后端地址（`http://localhost:5000`）域名或端口不一致。
- **解决方案**：
    1. 确认 `backend/app.py` 中已包含 `CORS(app)` 代码行（项目已包含）。
    2. 如果仍然报错，检查前端 `script.js` 中的请求 URL 是否写错。

### 🗺️ 4. 前端地图显示问题

#### Q9: 页面打开了，地图显示为空白/灰色的网格
- **原因**：高德地图 API Key 过期或失效，或者无法加载高德服务器资源（网络限制）。
- **解决方案**：
    1. 检查浏览器 F12 控制台（Console）是否有红色报错（如 `INVALID_USER_KEY`）。
    2. 如果 Key 失效，请前往 [高德开放平台](https://lbs.amap.com/) 申请新的 **Web端(JS API)** Key，替换 `frontend/index.html` 底部 script 标签中的 `key` 值。

#### Q10: 3D 地球不显示建筑，或者视角拉近后全是灰白的平面
- **原因**：卫星图服务加载异常。
- **解决方案**：
    - 在 `frontend/static/script.js` 中，找到初始化地图的代码，确保设置了 `features: ['bg', 'building', 'point']`。
    - 尝试将 `viewMode: '3D'` 确保开启。

#### Q11: 点击“开始分析”按钮没反应，F12 看到 `Uncaught ReferenceError: AMap is not defined`
- **原因**：高德地图 JS API 未加载完成就执行了 `script.js` 中的初始化代码。
- **解决方案**：
    1. 检查 `index.html` 中高德地图的 `<script>` 标签是否在 `script.js` 之前加载。
    2. 在网络较慢的情况下，可尝试在 `script.js` 中使用延时加载或 `window.onload` 包裹初始化函数。

### 📱 5. 边缘设备部署问题

#### Q12: 将代码拷贝到边缘端（如新大陆盒子/树莓派）运行 `python edge_infer.py` 报错 `Illegal instruction` 或 `Segmentation fault`
- **原因**：安装的 PyTorch 版本与该设备的 ARM CPU 架构指令集不兼容。
- **解决方案**：
    1. 卸载当前 PyTorch：`pip uninstall torch`
    2. 安装专门为 ARMv7l / aarch64 编译的 PyTorch 轮子（通常在官方 Forums 或 PiWheels 查找对应版本）。
    3. **终极方案**：在 PC 端完成模型导出（`torch.jit.trace`），然后在边缘端只安装轻量级的推理库 **ONNX Runtime**（无需安装完整的 PyTorch）。修改 `edge_infer.py` 使用 ONNX 推理。

#### Q13: 边缘设备推理速度极慢（>5秒/张）
- **原因**：未使用量化模型或硬件资源紧张。
- **解决方案**：
    1. 确保推理脚本加载的是 `model_quantized.pt`（INT8 量化后的模型），而不是原始的 FP32 模型。
    2. 检查边缘设备是否有其他进程占用 CPU，输入 `htop` 命令查看负载。

### 📁 6. 数据与文件问题

#### Q14: 找不到 `eurosat_resnet50.pth` 文件
- **原因**：未运行训练脚本或文件被误删。
- **解决方案**：
    1. 运行一次 `backend/model/train.py` 即可生成该文件。
    2. 或者修改 `app.py` 中的 `MODEL_PATH` 变量，指向一个不存在的路径，程序将自动进入 **Demo 演示模式**（随机返回结果用于测试前端）。

#### Q15: 上传图片后分析结果为“未知”或错误
- **原因**：图片预处理方式与模型训练时不一致。
- **解决方案**：
    1. 检查上传的图片格式（支持 `.jpg`, `.png`）。
    2. 如果是直接拍摄的手机照片（分辨率极高且色彩复杂），可能无法准确识别。建议使用从 Google Earth 截取的航拍视角图片进行测试。
